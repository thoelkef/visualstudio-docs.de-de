# <a name="c26138"></a>C26138
Warnung C26138: Das Anhalten einer Coroutine bei der Sperre \<Sperre >.

Warnung C26138 gibt eine Warnung aus, wenn eine Coroutine angehalten wird, während eine Sperre. Im Allgemeinen nicht wir wissen, wie lange eine Coroutine wird verbleiben im Zustand "angehalten", damit dieses Muster zu mehr kritische Abschnitte führen kann, als erwartet.

## <a name="example"></a>Beispiel
 Der folgende Code generiert C26138.

```cpp

#include <experimental/generator>
#include <future>
#include <mutex>

using namespace std::experimental;

std::mutex global_m;
_Guarded_by_(global_m) int var = 0;

generator<int> mutex_acquiring_generator() {
  global_m.lock();
  ++var;
  co_yield 1;                           // @expected(26138), global_m is hold while yielding.
  global_m.unlock();
}

generator<int> mutex_acquiring_generator_report_once() {
  global_m.lock();
  ++var;
  co_yield 1;                           // @expected(26138), global_m is hold while yielding.
  co_yield 1;                           // @expected(26138), global_m is hold while yielding.
  global_m.unlock();
}
```

## <a name="example"></a>Beispiel
 Der folgende Code wird diese Warnungen behoben.

```cpp

#include <experimental/generator>
#include <future>
#include <mutex>

using namespace std::experimental;

std::mutex global_m;
_Guarded_by_(global_m) int var = 0;

generator<int> mutex_acquiring_generator2() {
  {
    global_m.lock();
    ++var;
    global_m.unlock();
  }
  co_yield 1;                           // no 26138, global_m is already released above.
}
```
